# LLM API Connection

The OpenRouter API was accessed using the Python `requests` library.

The API key was stored securely using an environment variable.

python
os.environ["LLM_API_KEY"] = "YOUR_API_KEY"

No API keys were hardcoded anywhere in the notebook.

# call_llm() Function

A reusable `call_llm()` function was implemented.

The function:

- Creates the JSON request payload
- Adds authentication headers
- Sends an HTTP POST request
- Checks for HTTP status code 200
- Returns the generated LLM response

The function was successfully tested using the prompt:

# Why Temperature = 0?

Temperature was set to **0** because this project requires deterministic and reproducible JSON output.

Lower temperature always selects the highest probability token, producing more consistent structured responses.

# Temperature Comparison

| Temperature | Behaviour |
|-------------|-----------|
| **0.0** | Deterministic, consistent JSON output |
| **0.7** | More creative and variable responses |

### Observation

Temperature **0** produced consistent structured explanations, whereas **0.7** generated responses with more variation in wording while maintaining the overall meaning.

# JSON Schema

The expected JSON schema contained five required scalar fields.

python
{
    "prediction_label": "string",
    "confidence_level": "string",
    "top_reason": "string",
    "second_reason": "string",
    "next_step": "string"
}

The schema was validated using

python
jsonschema.validate()

Validation errors were caught using

python
ValidationError

If validation failed, a fallback dictionary containing null values was returned.

# PII Guardrail

Before every LLM request, a regular expression checked for personally identifiable information.

Blocked patterns:

- Email addresses
- Phone numbers

Example:

Input:
student@gmail.com

Output
Input blocked: PII detected.

Input
Age = 35
BMI = 28.4

Output
Safe input

The guardrail successfully blocked sensitive information before making API requests.
